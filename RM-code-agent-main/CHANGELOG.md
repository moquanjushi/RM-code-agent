# 变更日志

本项目遵循[语义化版本](https://semver.org/lang/zh-CN/)。

## [1.4.0] - 2024-01-15

### 新增
- **安全沙盒系统**: 集成Docker容器隔离执行环境
- **GitHub部署工具**: 自动化部署脚本和工作流
- **项目文档标准化**: 完整的贡献指南和部署规范
- **技能系统框架**: 支持代码审查和测试生成技能

### 改进
- **工具系统**: 所有执行工具现在在Docker沙盒中运行
- **错误处理**: 添加智能降级机制，Docker不可用时使用本地执行
- **代码质量**: 集成GitHub Actions自动化代码检查
- **安全性**: 添加安全扫描和依赖检查

### 修复
- **run_tests工具**: 修复沙盒执行问题
- **run_linter工具**: 修复代码检查工具集成

## [1.3.0] - 2024-01-10

### 新增
- **Web界面**: 基于Next.js的前端用户界面
- **实时进度跟踪**: 可视化任务执行进度
- **多轮对话支持**: 改进的上下文管理

### 改进
- **前端性能**: 优化界面响应速度
- **用户体验**: 改进交互设计和错误提示

## [1.2.0] - 2024-01-05

### 改进
- **工具扩展性**: 零代码工具配置
- **协议兼容性**: 支持标准MCP服务器

## [1.1.0] - 2024-01-01

- **代码分析工具**: AST解析和依赖分析
- **人机协作**: 计划确认和修改机制

### 改进
- **执行效率**: 优化步骤规划和执行

## 版本策略

### 版本号格式
- **主版本号**: 不兼容的API修改
- **次版本号**: 向下兼容的功能性新增
- **修订号**: 向下兼容的问题修正

### 发布周期
- **功能版本**: 每月发布一次（次版本号增加）
- **修复版本**: 根据需要发布（修订号增加）
- **重大版本**: 重大架构调整时发布（主版本号增加）

## 迁移指南

### 从1.3.0升级到1.4.0
1. 更新依赖包：`pip install -r requirements.txt`
2. 确保Docker已安装并运行
3. 运行部署脚本：`deploy_to_github.bat`

### 从1.2.0升级到1.3.0
1. 安装前端依赖：`cd frontend && npm install`
2. 更新后端API接口
3. 测试Web界面功能
